feat: Implement Lagos timezone enforcement and optimize database queries

This commit implements comprehensive Lagos timezone (Africa/Lagos, UTC+1) 
enforcement across the entire system and optimizes database queries to 
prevent timeouts on large tables.

## Timezone Implementation

### Frontend Components
- Updated critical components to use Lagos timezone for all date/time displays:
  * VehicleDetailsModal.tsx
  * RecentActivityFeed.tsx
  * OwnerChatDetail.tsx
  * VehicleTrips.tsx
  * TripHistoryTable.tsx
  * AlarmReport.tsx
  * ReportsSection.tsx
- Enhanced timezone utilities in src/lib/timezone.ts with formatLagos() helper
- All date displays now consistently show Lagos timezone (UTC+1)

### Database
- Created SET_DATABASE_TIMEZONE.sql to set database timezone to Lagos
- All database operations now use Lagos timezone context

## Invalid Timestamp Detection & Cleanup

### Detection Scripts
- FIND_INVALID_TIMESTAMPS.sql: Comprehensive detection with optimizations
- FIND_INVALID_TIMESTAMPS_FAST.sql: Fast version using EXISTS checks and sampling
- Both scripts optimized with time-based filters and LIMIT clauses to prevent timeouts

### Cleanup Scripts
- CLEANUP_INVALID_TIMESTAMPS.sql: Standard cleanup with time filters
- CLEANUP_INVALID_TIMESTAMPS_BATCH.sql: Batch processing (100 records per batch)
- CLEANUP_INVALID_TIMESTAMPS_ULTRA_FAST.sql: Ultra-optimized (100 records, CTEs, 7-day window)
- CLEANUP_SIMPLE_ONE_BY_ONE.sql: One-by-one processing with automated DO blocks

### Cleanup Strategies
- Option A: Set invalid dates to NULL (preserves records, recommended)
- Option B: Delete invalid records (aggressive)
- Option C: Update to current time (preserves records)

## Database Query Optimization

### Performance Improvements
- Added time-based filters to all queries (last 7-30 days instead of full table scans)
- Implemented LIMIT clauses to prevent scanning millions of rows
- Used CTEs instead of subqueries for better performance
- Created sampling strategies for large table queries
- Optimized verification queries with EXISTS checks
